Architecture
Two Governance Layers — Not One
Layer 1 — User Permissions: Exists in downstream systems. Access follows credentials. Agents inherit user permissions.
Layer 2 — Agent Governance: New. What Claude can do with those permissions — multi-system queries, action chaining, read/write, prompt injection. Guardrails separate from model logic.
↳ Connector walkthrough maps where Layer 1 ends and Layer 2 begins
Timeline
Platform Choice ≠ Speed
Native connectors solve authentication — OAuth, hours. But auth isn't the bottleneck. Server-side governance config is: tool scoping, DLP, cross-system boundaries, audit format. Identical work regardless of platform.
↳ Scope per-connector lift before committing to timelines
Sequencing
Compliance Defines the Standard
Technology builds the options. Compliance chooses. Going prepared with clear trade-offs produces a better outcome than going early with abstract questions. Everything we log can also be used against us.
↳ Build the technical picture first, then bring compliance real options
Non-Starter Without Controls
Cowork — Not for Regulated Workloads
Anthropic states: "Do not use Cowork for regulated workloads." Not in audit logs, Compliance API, or data exports. Compensating control: Cortex XDR with Koi integration provides endpoint-level visibility into AI agent behavior on the device.
↳ Disable until Koi ships in Cortex XDR or Anthropic builds audit coverage
Non-Starter
Excel & PowerPoint Add-Ins — Same Gap
Neither inherits data retention settings. Neither in audit logs or Compliance API. Both beta/research preview. Marketed to financial services while documentation says otherwise.
↳ Off the table for client data until audit logging confirmed in writing
Westfield-Specific
File Sync — Undetectable AI Changes
Box Sync and OneDrive: Claude edits files locally, changes sync to cloud attributed to the user. No system captures AI involvement. Downstream audit trails become actively misleading.
↳ Block desktop tools on synced folders. Cortex XDR/Koi provides interim detection.
Vendor
Marketing vs. Documentation
Claude for Financial Services launched with RBC, D.E. Shaw, Bridgewater. Every tool in that workflow is outside the audit perimeter. Marketing says financial services. Documentation says not for regulated workloads.
↳ Ask Anthropic what other regulated firms have in their contracts
Usable Today
Claude Web, API & Code — Audit-Ready
Web interface, API, and Claude Code are covered by Enterprise audit logs, Compliance API, custom retention, SSO/SCIM. These are the governed surfaces. The limitation is desktop tools — not Claude.
↳ Deploy through governed surfaces now
Licensing
Negotiate Audit Coverage in Contract
Audit coverage for Cowork, Excel, and PowerPoint should be a contract condition — not a roadmap item. Other regulated firms may have provisions not in the standard Enterprise plan.
↳ Make desktop tool audit coverage a contract requirement
Claude is ready for Westfield through governed surfaces. Desktop tools are not — and with Box Sync/OneDrive they make undetectable changes that corrupt existing audit trails. User permissions and agent governance are two separate layers. Deploy through governed channels. Negotiate desktop tool coverage in the Enterprise contract. Move fast — but build it right.

03 · Airia Governance Layer

The governance
platform assessed

Airia validated as the governance layer on a March 2026 demo call. Here's what it provides, what Westfield still needs to define, and how it fits the two-layer audit model.

Confirmed — Demo Call
What Airia Provides
MCP Gateway
~1,100 pre-catalogued MCP servers. Admins approve which servers are available org-wide. Snowflake access restrictable to specific schemas/tables. Containerized MCP server hosting.
AI Gateway
Full request/response logging across all LLM traffic. Guardrails for prompt injection, sensitive data exposure, policy violations. Per-user and per-gateway dollar-value budgets with automatic routing to cheaper models.
Surfaces
Claude Code and Cowork: full control via gateway routing. Web app: browser extension monitoring. Native desktop app: cannot currently be intercepted — compliance API approach on short-term roadmap.
Multi-Model
Not Claude-exclusive. Supports OpenAI, Gemini, self-hosted models on AWS/Azure. Relevant if Westfield diversifies models.
Standalone
MCP Gateway works without Claude Enterprise. Full org-wide policy enforcement through Claude requires Enterprise licensing.
Must Configure
What Westfield Still Has to Define

Airia provides the platform. Someone still has to define the rules. This is the Layer 2 agent governance work.

Agent-to-system access policies — which agents can access which systems. Per-role, per-team, per-workflow.
DLP rules and data classification — what data can leave each system and reach the model. PII, client account numbers, sensitive fields.
Cross-system boundary policies — rules for when a single prompt touches multiple systems.
Approval workflows — which agent actions require human confirmation before execution.
Risk classification — tag agents, models, and data sources with Westfield's risk definitions.
Routing rules — route requests based on data sensitivity, team, and compliance requirements.
Pricing
Airia Pricing Structure
Tiers
Free (eval, 1 user), Individual ($50/mo), Team ($250/mo), Enterprise (custom — quote-based).
Westfield
Enterprise is the only relevant tier. SSO/SAML, audit logs, flexible deployment, custom SLAs, dedicated support.
Trial
14-day free trial available. Use to validate governance controls, connector setup, and latency before commitment.
Cost Model
Airia platform fee + Claude token costs. Confirm whether token costs are additive or inclusive. Key negotiation point.
Architecture Fit
How Airia Fits the Two-Layer Model
Layer 1
User permissions stay downstream. Airia passes user identity through to each system. No change from current state.
Layer 2
Airia IS the agent governance layer. Sits between Claude and every downstream system, enforcing DLP, boundaries, tool scoping, audit logging.
Cortex XDR
Complementary, not competing. Airia governs MCP/API layer. Cortex XDR/Koi governs endpoint. Belt and suspenders.
Desktop Gap
Three paths: (1) Anthropic native audit, (2) Cortex XDR/Koi at endpoint, (3) Airia compliance API approach. Desktop tools blocked until at least one ships.

Token rates (per MTok): Sonnet 4.6: $3 input / $15 output. Opus 4.6: $5 input / $25 output. Prompt caching: 90% discount on cached input (could reduce costs 30–50%).

Usage basis: 65 users firm-wide, 22 working days/month. Phase 1: ~30% heavy, 40% moderate, 30% light. Phase 2: 30–50 complex agentic queries/day.

Snowflake context: Westfield's Snowflake spend has already grown from $5K to $20K/month on human-driven queries. AI queries at machine speed will accelerate this. Dedicated VW with cost alerts is non-negotiable.

Biggest swing: Data retrieval volume per agentic query. A single Snowflake query pulling 200K tokens costs $0.60–$1.00 in input alone. 50 queries/day = $660–$1,100/month just from Snowflake retrieval.

Productivity Frame
65 users recovering 3–5 hours/week = 195–325 hours of capacity per week without adding headcount. At $7K–$16K/month, that's $2.15–$4.94 per recovered hour. Payback is measurable within the pilot window.
