01 · The Core Tension

The mandate changed
while the project was in flight

"WIP 2.0 was the right answer to the question Westfield was asking 18 months ago. How WIP 2.0 feels legacy before it launches."

Dee Silveira · CTO, Westfield Capital Management · March 2026

This isn't a failure of the WIP 2.0 project. It's a signal that the intelligence layer changes the fundamental question. WIP 2.0 was designed to answer: "How do we report better?" Claude answers: "How do we reason together?"

02 · Side by Side

What each approach
actually delivers

WIP 2.0 — The Reporting Layer
Core question
How do we show what happened?
Better dashboards, cleaner data presentation, improved Sigma visualizations of portfolio activity.
User interaction
View pre-built reports
Analysts navigate to fixed views. Answers are designed in advance by whoever built the dashboard.
Data freshness
Scheduled refresh cycles
Reports update on a cadence. Users work with data as of the last refresh — not necessarily now.
Insight generation
Human interprets the chart
The system surfaces data. The analyst decides what it means and writes their own commentary.
Personalization
Filtered views per user
Some role-based filtering. Fundamentally the same report for everyone with slight variations.
Output format
Dashboard or exported PDF
Static snapshots. Requires human effort to turn into a client-ready communication.
vs
Claude Intelligence Layer — The Reasoning Layer
Core question
What does this mean, and what should we do?
Natural language queries against live data. Surfaces patterns, anomalies, and implications — not just numbers.
User interaction
Ask any question in plain language
"Why did attribution shift this week?" "How does this compare to our thesis?" No pre-built view required.
Data freshness
Live query via MCP to Snowflake
Every response reflects current data. The answer is always as fresh as the underlying source.
Insight generation
Claude synthesizes and explains
The system generates the narrative, not just the chart. Attribution commentary written automatically.
Personalization
Fully contextual per request
Every output is generated fresh for the specific question, client, or context — zero marginal cost per variation.
Output format
HTML, prose, or structured data
Client-ready output generated directly. Presentation layer and analysis layer unified in one step.

03 · What This Changes

Not a better dashboard.
A different category.

🔄
Reporting → Reasoning
WIP 2.0 makes it faster to see the data. Claude makes it possible to understand it. These are different products serving different cognitive needs.
📐
Fixed views → Fluid queries
Every WIP dashboard represents a decision made in advance about what matters. Claude lets the question drive the answer — not the other way around.
✍️
Charts → Commentary
The bottleneck was never producing the chart. It was writing the attribution sentence below it. Claude eliminates that bottleneck entirely.
🎯
One report → Infinite variations
WIP serves the firm. Claude serves each relationship — client, PM, IC, board — with a contextually appropriate response at zero marginal cost.

04 · The Strategic Call

What happens to WIP 2.0?

Strategic verdict

WIP 2.0 and Claude are not competitors — they operate at different layers. WIP 2.0 remains the operational data foundation. Claude becomes the intelligence layer on top of it. The question is sequencing, not replacement.

A
Pause WIP 2.0 — Complete Claude+MCP first
Sprint through the intelligence layer. Use Claude's output to re-examine what WIP 2.0 actually needs to be — potentially a much simpler data layer feeding Claude rather than a full reporting product.
B
Reframe WIP 2.0 as the data layer
Scope WIP 2.0 down to what Claude needs: clean, structured, reliable data in Snowflake. Remove the presentation layer ambitions — Claude handles that now.
C
Complete WIP 2.0 as scoped — Integrate Claude as Phase 2
Deliver on current commitments, then layer Claude on top in Q3-Q4. Slower path to full intelligence capability, but maintains delivery predictability.

05 · The Sigma Question

Where does Sigma fit?

Sigma was the visualization layer for WIP 2.0. With Claude generating narrative output directly, the role of a BI visualization tool shifts. Sigma becomes most valuable as an ad-hoc exploration tool for analysts — not as the primary client-facing output mechanism.

Sigma keeps
Ad-hoc analyst exploration, complex custom visualizations, data discovery, pivot analysis that falls outside Claude's scope.
Claude replaces
Standard client reports, attribution commentary, scheduled outputs, any visualization that can be described in prose or rendered in HTML.
Both together
Analyst uses Sigma to explore, finds a pattern, asks Claude to write the client communication. The tools complement rather than compete.
