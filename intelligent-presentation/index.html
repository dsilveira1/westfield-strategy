02 · The Shift

Reporting vs. Reasoning

Today — Static Production

  • Analyst pulls data from Snowflake/Eze manually
  • Formats into PowerPoint template
  • Reviews, revises, exports to PDF
  • Emailed as attachment — immediately static
  • Stored forever as another version file
  • Repeated in full for every client variation
  • No personalization at scale
  • Zero interactivity for the reader

Future — Intelligent Generation

  • Claude queries Snowflake live via MCP
  • Generates HTML presentation from prompt + template
  • Personalized per client at zero marginal cost
  • Delivered as a link — always regenerable
  • Store the prompt + template, not 500 PDF copies
  • Interactive charts, filters, drill-downs included
  • Analyst reviews output, not production process
  • Regenerate in seconds when data updates

05 · Implementation Path

Bolt-on to the
existing sprint

1
Template Library
Define 3–5 canonical HTML presentation templates (client review, prospect pitch, internal report, IC package). Establish Westfield brand standards in CSS. Non-technical staff never touch these.
Parallel to Phase 2 Engineering Sprint
2
Pilot — Internal Weekly Report
Start with lowest-stakes, highest-frequency output: internal performance summary. Prove generation quality and time savings before client-facing use.
Phase 3 Resource Sprint · Validation Gate
3
Delivery Infrastructure
Determine delivery method per output type: email attachment (HTML file), shared drive link, intranet URL, or PDF export via browser print. Align with compliance on archival requirements.
Phase 3 · Compliance & IT Alignment
4
Client-Facing Rollout
Expand to quarterly client reviews and prospect materials. Measure analyst time saved, storage delta, and qualitative feedback on output quality vs. legacy decks.
Phase 4 Connect Sprint · Q3 Milestone
